# THREAT-SIM

THREAT-SIM is an academic ransomware-impact demonstration project for the VIT Information Security Management course. It simulates a ransomware incident inside a controlled environment, measures how different security postures change the outcome, and presents those results in a web dashboard.

There is no real malware. Encryption is real, but it is applied only to synthetic files generated by the system inside the backend workspace.

## Tech Stack

- Frontend: React 18 + Vite
- Backend: FastAPI
- Real-time updates: WebSocket
- Encryption: `cryptography` / Fernet
- Process inspection: `psutil`
- Concurrency: Python `threading`

## How The Demo Works

1. The user chooses one of four organizational security strategies in the frontend.
2. The frontend calls `POST /simulate`.
3. The backend creates a clean synthetic file system in `backend/sim_workspace/org_files/`.
4. Defense setup runs according to the selected strategy.
5. The ransomware simulation encrypts files one by one in a worker thread.
6. For strategies 3 and 4, a watcher thread monitors file modification bursts and signals the attack to stop.
7. Recovery logic runs.
8. The assessment module computes final metrics.
9. The frontend receives live logs over WebSocket and renders the final dashboard.

## Run In Development

### Backend

From the project root:

```bash
python -m pip install -r backend/requirements.txt
uvicorn backend.app:app --reload
```

The backend runs on `http://localhost:8000`.

### Frontend

In a second terminal:

```bash
cd frontend
npm install
npm run dev
```

Open `http://localhost:5173`.

## Production-Style Local Run

To serve the built frontend through FastAPI:

```bash
cd frontend
npm run build
xcopy /E /I /Y dist\* ..\backend\static\
```

Then start:

```bash
uvicorn backend.app:app --reload
```

The backend will serve the built frontend from `backend/static/`.

## API Surface

- `POST /simulate`
- `GET /strategies`
- `GET /results/{simulation_id}`
- `WebSocket /ws/{simulation_id}`

## Important Safety Note

This project is intended only for a controlled academic VM or isolated lab environment. Do not run it on a production machine or on any network connected to real organizational systems.

## Faculty Demo Summary

The cleanest way to explain this project is:

- `frontend/` visualizes the attack lifecycle and final outcomes.
- `backend/` performs the actual controlled simulation and metric computation.
- Strategies 1 and 2 use projected downtime values.
- Strategies 3 and 4 use measured detection and recovery behavior.
